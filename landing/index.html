生产级特性

每个链的 compose 编排都内置了以下能力

✓

健康检查

所有服务配置 Docker healthcheck，自动检测 RPC 可用性并联动重启策略。

⛶

资源限制

deploy.resources 限制 CPU / 内存，防止单节点耗尽宿主机资源。

◷

日志轮转

json-file 驱动 + 自动轮转（100MB × 3），避免磁盘被日志写满。

$

环境变量驱动

L2 链（Arbitrum、Scroll）外部依赖通过 .env 注入，不硬编码地址。

↻

统一重启策略

全部使用 unless-stopped，非预期退出自动拉起，手动停止保持停机。

⬇

快照加速

支持通过 publicnode 快照快速同步，减少从零追赶区块的等待时间。

快速开始

三步启动一个全节点

# 1. 克隆仓库
git clone git@github.com:orange723/deploy-chain-fullnode.git
cd deploy-chain-fullnode

# 2. 进入目标链目录并初始化
cd eth
./init-eth.sh

# 3. 启动
docker compose up -d
