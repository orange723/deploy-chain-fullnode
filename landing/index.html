生产级特性

每个 compose 编排都内置了以下能力

健康检查

所有服务配置 Docker healthcheck，自动检测 RPC 可用性并联动告警与重启策略。

资源限制

deploy.resources 限制 CPU 与内存上限，防止单容器耗尽宿主机，多链并行部署互不干扰。

日志轮转

json-file 驱动 + 自动轮转（100MB × 3），不再担心容器日志写满磁盘。

环境变量驱动

L2 链（Arbitrum、Scroll）通过 .env 管理外部依赖，配置与代码分离，部署无需改 compose。

统一重启策略

全部使用 unless-stopped，异常退出自动拉起，手动停止保持停机，行为一致可预期。

快照加速

支持 publicnode 快照导入，大幅缩短首次区块追赶时间，从数天降至数小时。

快速开始

前提：Docker ≥ 24.0，Docker Compose ≥ v2.20

终端 
# 1. 克隆仓库
git clone git@github.com:orange723/deploy-chain-fullnode.git
cd deploy-chain-fullnode

# 2. 初始化目标链
./init.sh eth

# 3. 进入目录并启动
cd eth && docker compose up -d

# 4. 查看同步进度
docker compose exec geth geth attach --exec 'eth.syncing' http://localhost:8545
